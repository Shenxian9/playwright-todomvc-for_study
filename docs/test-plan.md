# 测试计划（Test Plan）

## 1. 测试目标
- 验证 TodoMVC 核心交互在主流浏览器下可稳定运行。
- 建立可复用、可维护的 Web 自动化测试实践样板。
- 形成可用于实习汇报的标准化测试文档与产物。

## 2. 测试范围
### 包含
- 新增/完成/删除/筛选等核心功能
- 空输入、重复输入、长文本输入等边界场景
- 刷新后的状态持久化
- 基础冒烟流程

### 不包含
- 后端接口测试
- 性能压测
- 安全渗透测试
- 数据库或服务端一致性验证

## 3. 测试环境
- Node.js 20+
- Playwright Test
- 浏览器：Chromium、Firefox
- 目标站点：TodoMVC（Ember.js）

## 4. 测试策略
- 使用 POM 封装页面操作与常见断言。
- 用例按 functional / boundary / persistence / smoke 分类。
- 失败自动保留 screenshot 与 trace。
- 通过 CI 执行回归并归档报告。

## 5. 风险点
- 外部公开站点偶发网络波动可能影响稳定性。
- 目标站点升级导致页面结构变化，可能引发定位器失效。
- 跨浏览器表现存在细微差异，需结合产物进一步分析。

## 6. 通过准则
- 冒烟测试 100% 通过。
- 功能与持久化测试全部通过。
- 边界测试结论与页面真实行为一致，断言无臆造需求。
- 报告与失败产物可正常生成并追溯。
