# 实习/课程汇报提纲（Report Outline）

## 1. 项目背景
- 选择 TodoMVC 作为公开、可复现实验对象
- 目标是完成 Web 自动化测试实践闭环
- 明确项目性质为“开源项目复现与扩展”

## 2. 测试需求分析
- 识别核心用户操作路径（新增、完成、删除、筛选）
- 拆分基础功能、边界场景与状态持久化场景
- 定义冒烟回归最小集

## 3. 测试用例设计
- 功能测试用例设计原则
- 边界测试用例设计原则
- 持久化与稳定性验证思路

## 4. 自动化实现
- Playwright + TypeScript 技术选型
- Page Object Model 封装策略
- 测试数据分层与目录结构设计
- CI 自动化执行与产物归档

## 5. 执行结果
- 通过率统计
- 失败用例定位方式（截图、trace）
- 报告展示与结果解读

## 6. 问题与优化方向
- 当前覆盖边界与限制
- 可扩展项（如编辑、批量完成、跨端测试）
- 后续改进计划（稳定性、可维护性、覆盖率）
